# Space Issue Report Prototype

A browser-based iPad prototype for magicplan field workers. The app allows a contractor on-site to report a space issue and send a structured async handoff to a remote expert without needing a live call.

Built as a take-home design challenge for the Product Designer role at magicplan.

## Tech stack

- React 18
- TypeScript
- Vite
- Tailwind CSS v4
- lucide-react
- Local React component state only

## Run locally

Requirements:

- Node.js 18 or newer
- npm

Clone or download the repository, then run:

npm install

npm run dev

Open the local URL printed in the terminal.

The default Vite URL may be:

http://localhost:8080

## Production build

Run:

npm run build

npm run preview

## Prototype flow

The prototype covers the complete async issue-reporting flow from job context to simulated expert handoff.

### 1. Job overview

The contractor is mid-capture at Bauer Residence.

The overview shows:

- Current project context
- Existing 2023 floor plan reference
- Remote expert availability
- A clear “New space issue” action

### 2. Problem type

The contractor selects the type of issue:

- Wall missing from plan
- Measurement off
- Floor plan mismatch
- Other

The default walkthrough uses “Wall missing from plan”.

### 3. Location

The contractor marks the affected area on a simplified SVG version of the 2023 floor plan.

The location step includes:

- Tappable room areas
- Issue pin placement
- Selected room context
- A precise location reference, such as the south wall near the hallway opening

### 4. Evidence

The contractor adds the context that applies to the issue.

The evidence step includes:

- Simulated photo attachment
- Written note
- Simulated voice memo option
- Quick tags
- Optional measurement

The prototype includes realistic pre-filled content so the complete flow can be reviewed without extensive typing.

### 5. Structured summary

The app generates a structured local report from the contractor’s inputs.

The summary includes:

- Problem
- Location
- Evidence
- Impact
- Requested decision
- Urgency

The summary is generated by a deterministic local function, not by an external AI service.

Every field can be reviewed and edited before continuing.

### 6. Review

Before sending, the contractor checks that the required context is complete.

The review step includes:

- Validation checklist
- Urgency selection
- Time-sensitive context
- Expert preview
- A clear view of exactly what the remote expert will receive

The expert preview shows:

- What is wrong
- Where the issue is
- Which evidence is attached
- What decision is requested
- Why the issue is urgent

### 7. Sent confirmation

The final screen simulates successful delivery to the remote expert.

It confirms:

- The issue has been sent
- The expert has been notified
- The expert is available until 15:00 CET
- The contractor can continue capturing other rooms
- A notification will appear when the expert confirms the correction


## Scope

This is a front-end prototype.

Included:

- Interactive multi-step flow
- Local React state
- Simplified SVG floor plan
- Issue pin placement
- Simulated photo attachment
- Simulated voice memo option
- Quick tags and optional measurement
- Deterministic local summary generation
- Editable report fields
- Expert preview
- Simulated send confirmation
- Responsive layout optimized for iPad-sized viewports

Not included:

- Backend
- Authentication
- Database
- Real file storage
- Real voice recording
- External AI API
- Real message delivery
- Environment variables

The local summary is generated by a deterministic function in:

src/utils/generateSummary.ts

No external AI service or API key is required.

## Reviewer notes

- No setup beyond npm install is required.
- No API keys or environment variables are required.
- All prototype data stays in the browser.
- The app is optimized for iPad portrait and landscape, but can also be reviewed in a desktop browser.

## Troubleshooting

- If dependencies fail to install, verify Node.js 18 or newer.
- If the default port is already in use, use the alternative URL printed by Vite.
- If the preview does not open automatically, copy the terminal URL into the browser.
